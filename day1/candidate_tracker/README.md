# Candidate Tracker

## Contexte

Tu travailles chez Talya. Les recruteurs reçoivent des candidatures et tu dois développer un petit programme pour gérer ces candidats.

Tout se passe en mémoire, sans base de données.

## Objectifs

- Utiliser des interfaces et des types
- Manipuler des tableaux d'objets
- Utiliser `map`, `filter`, `reduce`, `find`, `some` et `every`
- Écrire des fonctions pures
- Préserver l'immutabilité
- Utiliser le destructuring, l'optional chaining, les union types et des propriétés optionnelles
- Garder une organisation propre du code

## Fonctionnalités

- Créer un candidat
- Supprimer un candidat
- Modifier le statut
- Ajouter une compétence
- Ajouter une note
- Trouver un candidat par id
- Trouver tous les candidats `accepted`
- Trouver tous ceux ayant React
- Trouver tous ceux ayant plus de X années d'expérience
- Vérifier s'il y a au moins un candidat accepté
- Vérifier que tous les candidats ont une adresse mail
- Vérifier s'il existe un candidat sans compétence
- Compter le nombre de candidats
- Calculer le salaire moyen demandé
- Calculer le salaire maximum
- Calculer le salaire minimum
- Compter les candidats par statut
- Identifier la compétence la plus présente
- Calculer la moyenne d'expérience
- Afficher un candidat proprement avec `displayCandidate()`
- Afficher les statistiques avec `displayStatistics()`

## Contraintes

- Pas de boucle `for`
- Pas de boucle `while`
- Utiliser uniquement `map`, `filter`, `reduce`, `find`, `some` et `every`
